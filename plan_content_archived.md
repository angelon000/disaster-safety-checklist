# 코리아콘텐츠랩(KCL) 맞춤형 체크리스트 시스템 (아카이브)

> **⚠️ 주의**: 이 문서는 과거 콘텐츠 사업 도메인용 체크리스트 시스템 초안입니다.
> 현재 시스템은 **재난·안전 도메인**으로 전환되었으므로, 이 문서는 참고용 아카이브로만 보관합니다.
> 실제 구현 계획은 [plan.md](plan.md)를 참조하세요.

---

## 1. 목표 및 개요

**대상**: 콘텐츠 서비스/플랫폼/솔루션을 준비하는 KCL 고객(스타트업·기업·기관)

**목적**: 아이템별로 필요한 "시장·정책·기술·사업화" 점검 항목을 자동/반자동으로 생성하고, 각 항목 뒤에 참고할 웹자료·논문·기술·API 근거를 붙이는 것

**핵심 차별화**: 웹·논문·기술·무료 API 연구를 통한 데이터 기반 체크리스트 자동 생성

---

## 2. 체크리스트 카테고리 구조

| 카테고리 | 주요 점검 항목 |
|---------|--------------|
| **시장·수요** | 타깃 고객, 문제 정의, 수요 증거, 대체재 분석 |
| **콘텐츠·IP** | 저작권·2차 저작물·IP 계약, 해외 전개 시 이슈 |
| **경쟁·벤치마킹** | 유사 서비스, 차별화 요소, 가격·수익모델 비교 |
| **기술·제품** | 핵심 기술 스택, 구현 난이도, 성능·보안·확장성 위험 |
| **비즈니스모델·수익화** | 과금 구조, LTV·CAC, 파트너십 구조 |
| **마케팅·유통** | 유입 채널, 플랫폼 정책(스토어, OTT 등), SEO/ASO |
| **정책·규제·지원사업** | 국내·해외 규제, 정부·지자체 지원사업, 인증 |
| **운영·조직·리스크** | 인력·역할, 운영 코스트, 법무·세무·리스크 관리 |

## 3. 데이터 수집 모듈 설계

### 3.1 웹 리서치 모듈

**수집 소스**:
- 정부·공공: 문화체육관광부, KOCCA, KISA, KISDI, 통계청, 공공데이터포털
- 산업·시장: 산업리포트, 리서치 기관 블로그, 뉴스·보도자료
- 커뮤니티: 브런치, 미디엄, 기술 블로그, 스타트업 커뮤니티

**동작 방식**:
1. 키워드 기반 검색 (예: "웹툰 플랫폼", "K-POP 팬덤 서비스")
2. 상위 N개 링크 크롤링/스크랩
3. 제목·요약·발행일·출처·링크 저장
4. 간단 요약 생성 (LLM 기반)
5. 체크리스트 항목에 "참고 링크"로 연결

### 3.2 논문 리서치 모듈

**수집 소스** (무료 API):
- Semantic Scholar API
- CrossRef API
- arXiv API
- 국내: KCI (라이선스 이슈 고려, 키워드·링크 수준)

**동작 방식**:
1. 한글+영문 키워드 검색 (예: "metaverse concert", "AI drama recommendation")
2. 제목, 저자, 연도, 초록 요약, 주제 태그 저장
3. 기술·시장 성숙도 판단 근거 제공 (논문 수, 최신 연도 비중)

### 3.3 기술 트렌드·레포지토리 모듈

**수집 소스**:
- GitHub Search API (키워드, 스타 수, 업데이트 날짜)
- 패키지 레지스트리: npm, PyPI, Maven
- 앱/서비스: Product Hunt, App Store/Play Store

**동작 방식**:
1. 사업 아이템 키워드로 관련 오픈소스·라이브러리·샘플 프로젝트 수집
2. 기술 성숙도 지표 정의 (GitHub 스타, 최근 커밋, 포크 수)
3. "활용 가능 기술 스택 후보" 제시
4. 구현 난이도·리스크 항목과 연결

### 3.4 무료/공공 API 리서치 모듈

**수집 소스**:
- 한국 공공데이터포털 (api.data.go.kr) - 문화·관광·지역·인구·소비 관련
- 국제 오픈데이터: UN, OECD, World Bank
- 기타: YouTube Data API, News API (무료 플랜 범위 확인)

**동작 방식**:
1. 사업 아이템 관련 공공·오픈 API 목록 생성
2. 각 API별: 제공 데이터, 이용조건, 호출 예시 정리
3. "데이터·API 활용 가능성" 질문과 추천 API 목록 제시

## 4. 체크리스트 생성 엔진

### 입력
- 사업 아이템 키워드 (예: "K-POP 팬덤 플랫폼")
- 콘텐츠 유형 (게임, 웹툰, OTT, 메타버스, 교육, 음악 등)
- 사업 단계 (아이디어, PoC, 런칭, 성장, 해외진출)
- 관심 영역 가중치 (시장 중심/기술 중심/정책 중심)

### 처리 로직
1. 사업 유형·단계에 따라 기본 템플릿 질문 세트 선택
2. 웹·논문·기술·API 리서치 결과 조회 및 질문별 매핑
3. 자료 밀도에 따라 질문 중요도/우선순위 점수 부여
4. 부족한 영역은 "추가 탐색 필요" 플래그 표시

### 출력
- 구조화 체크리스트 (JSON/Markdown)
- 질문, 선택지/서술형, 중요도, 추천 참고자료
- KCL 컨설턴트 메모 공간
- 사업계획서 자동작성 연동 가능

## 5. 구현 로드맵

### 1단계: 템플릿화 및 수동 검증
- KCL 내부 체크리스트 문서를 템플릿화 (엑셀/Markdown)
- 웹·논문·기술 사례를 수동으로 붙여보면서 질문 구조·카테고리 확정

### 2단계: 데이터 수집 모듈 구현
- Python 스크립트로 각 무료 API에서 JSON 수집
- Semantic Scholar, CrossRef, arXiv, GitHub, 공공데이터 등
- 로컬 저장 및 데이터 정규화

### 3단계: 매핑 및 우선순위 로직
- 수집 데이터에 태그 추가 (도메인·키워드·연도·신뢰도)
- 점수/우선순위 계산 로직 구현
- 체크리스트 템플릿 매핑 규칙 설계

### 4단계: UI/CLI 구현
- 웹 UI 또는 CLI 도구로 "사업 아이템 입력 → 맞춤 체크리스트 + 참고자료 출력"
- 사업계획서 자동작성 파이프라인 연동

## 6. 사용 시나리오 예시

**시나리오**: "K-웹툰 글로벌 플랫폼" 신규 프로젝트

1. 컨설턴트가 키워드·대상 시장(EU/미국/일본)·사업 단계(해외진출) 입력
2. 시스템이 웹·논문·기술·API 조회
   - 글로벌 웹툰 서비스 분석
   - 관련 저작권 이슈 논문
   - 번역·추천 엔진 오픈소스
   - 문화·언어 관련 공공데이터 API
3. 체크리스트 자동 생성 및 참고자료 링크
4. 컨설턴트와 고객이 체크리스트 작성
5. 사업계획서 초안 자동 생성 (시장분석·기술·진출전략)

## 7. 기술 스택 (제안)

**백엔드**:
- Python 3.9+
- FastAPI (API 서버)
- SQLite/PostgreSQL (데이터 저장)
- Celery (비동기 수집 작업)

**데이터 수집**:
- requests, beautifulsoup4 (웹 크롤링)
- semanticscholar, arxiv (논문 API)
- PyGithub (GitHub API)
- pandas (데이터 처리)

**프론트엔드** (선택):
- React/Vue.js (웹 UI)
- 또는 CLI (Click/Typer)

**문서 생성**:
- python-docx (Word 문서)
- Markdown (간단 출력)
